10 
11 ## Features
12 
13 * Fully **portable**, **serializable** quest system, perfect for use in savegames. Supports JSON and a compact binary format.
14 * Separates quest logic from quest data (such as titles/descriptions) by using **template files**.
15 * Can output the quest properties as simple **text or HTML form** to support rich formatting in the game.
16 * Uses a complex **weighted graph search** to create new quests.
17 This results in more interesting and coherent quest stories as world actors and entities can be reused when fitting.
18 * Supports **mod directories** to overwrite quest templates after deployment.
19 * The game world entities to be used in quests are supplied directly from a game's custom world model.

25 ## Usage
26 
27 The quest system of a game is not an isolated part of the system as it is heavily involved in the current world state and events.
28 Therefore, the game has to provide the quest system with the necessary information about the game's state and has to
29 fulfill change requests made by the quest system (otherwise the state will be inconsistent).
30 
31 To use the QuestWeaver system, you have to follow these steps:
32 
33 * Implement and register custom `TemplateFactory` classes that read your custom template files
34 * Implement a custom `WorldModel` class
35 * Register your wold model and template factories with the `QuestWeaver` instance
36 
37 ```cpp
38 QuestWeaver Init() {
39  // Create your template factories
40  shared_ptr<QuestTemplateFactory> questFactory = make_shared<MyQuestTemplateFactory>();
41  shared_ptr<StoryTemplateFactory> storyFactory = make_shared<MyStoryTemplateFactory>();
42 
43  // Create Configuration
44  WeaverConfig config;
45  config.worldModel = make_unique<MyWorldModel>();
46  config.dirs.modDirectory = "./Template/";
47  shared_ptr<WorldListener> myWorldListener = make_shared<MyWorldListener>();
48 
49  // Create the quest system
50  QuestWeaver weaver(config);
51  weaver.RegisterQuestTemplateFactory(questFactory);
52  weaver.RegisterStoryTemplateFactory(storyFactory);
53  weaver.GetWorldModel().AddListener(myWorldListener);
54 
55  return weaver;
56 }
57 
58 // Create new quests
59 shared_ptr<Quest> newQuest = weaver.CreateNewQuest();
60 ```
61 
62 
63 Quests usually want to change the world state as well as their own state (e.g. when a quest succeeds).
64 To do that, you can tick the quest system in your game loop and it will take care of the rest by ticking each quest
65 separately.
66 The `WorldListener` allows you to change the game state whenever the world model was changed by the quest system.
67 
68 ```cpp
69 while (true) {
70  // game loop ...
71 
72  weaver.Tick(delta); // updates the world state
73 }
74 ```
75 
76 
77 You can also fully serialize the quest system, e.g. to create a save game or to debug it:
78 
79 ```cpp
80 void SerialTest() {
81  QuestWeaver weaver = Init();
82 
83  // serialize
84  stringstream ss; // can also be a file stream
85  weaver.Serialize(ss, StreamType::JSON);
86  cout << ss.str() << endl; // or save it to disk
87 
88  // deserialize - it is important to re-register the template factories and the world model listener!
89  QuestWeaver deserialized = QuestWeaver::Deserialize(ss, StreamType::JSON, config.dirs);
90  weaver.RegisterQuestTemplateFactory(questFactory);
91  weaver.RegisterStoryTemplateFactory(storyFactory);
92  weaver.GetWorldModel().AddListener(myWorldListener);
93 }
94 ```
